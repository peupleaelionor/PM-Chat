## Section : Pourquoi PM-Chat

### Titre de section

> Un système conçu pour l'autonomie totale de communication.

### Points clés

- **Communication directe entre appareils** — Les messages transitent d'appareil à appareil, sans serveur intermédiaire, sans relais cloud.

- **Réseau privé autonome** — Le réseau PM-Chat est fermé par conception. Seuls les appareils provisionnés avec le même code réseau peuvent communiquer.

- **Sans infrastructure externe** — Pas d'Internet, pas de Wi-Fi, pas de carte SIM, pas d'antenne-relais. Le système fonctionne partout où se trouvent les appareils.

- **Chiffrement AES-256-GCM local** — Chaque message est chiffré sur l'appareil avant transmission. Les clés restent sur l'appareil. Le chiffrement est structurel, pas optionnel.

- **Relais maillé multi-sauts** — Les appareils intermédiaires relaient automatiquement les messages, étendant la portée du réseau sans configuration supplémentaire.

- **Conçu pour minimiser l'exposition, la dépendance et la traçabilité** — Aucun serveur tiers, aucun compte utilisateur, aucun historique centralisé. Le système est conçu pour limiter ce qui peut être observé, intercepté ou exploité.

## Section : Comment ça fonctionne

### Titre de section

> Trois étapes. Un réseau opérationnel.

### Étapes

**1. Provisionner**

Configurer chaque appareil avec un code réseau partagé. Ce code définit le groupe de communication. Seuls les appareils partageant le même code peuvent échanger des messages.

**2. Communiquer**

Envoyer et recevoir des messages chiffrés via LoRa. L'interface est simple : composer, envoyer, lire. Le chiffrement est automatique et transparent pour l'utilisateur.

**3. Relayer**

Les appareils intermédiaires étendent automatiquement la portée du réseau. Un message peut traverser plusieurs appareils avant d'atteindre son destinataire, sans intervention manuelle.
